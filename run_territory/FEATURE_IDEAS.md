### 기능 1. 영역 합산 및 침범 시스템 (Territory Merge & Invasion)

**우선순위:** High
**예상 복잡도:** High (Pro 기능 — 서버 연동 필요)

#### 사용자 스토리

> 나는 달리기 유저로서, 이미 내가 확보한 영역 옆에 새 루프를 완성했을 때 두 영역이 자동으로 합쳐지기를 원한다. 또한 다른 유저가 내 영역과 겹치는 루프를 완성했을 때 영역이 충돌하고, 내가 더 큰 루프를 달려서 빼앗긴 영역을 되찾는 전략적 재미를 경험하고 싶다.

#### 기능 상세

- **로컬(Free):** 자신의 기존 영역과 새 영역이 인접하거나 겹칠 경우 자동 병합. 현재 `GeoUtils.mergePolygons()` (Convex Hull 기반)가 이미 구현되어 있으므로 `ClaimTerritoryUseCase`에서 중첩 감지 후 병합 호출만 추가하면 됨.
- **Pro(멀티플레이어):** 서버에서 다른 유저 영역과의 중첩 감지. 나중에 달린 유저가 이전 유저의 영역을 빼앗음 (Last-claim-wins). 빼앗긴 유저에게 푸시 알림 전송.
- **UI:** 지도 화면에서 충돌된 영역은 점선 테두리로 강조 표시. 자신의 영역은 실선, 빼앗긴 구역은 회색으로 페이드아웃 애니메이션.

#### 수락 기준 (Acceptance Criteria)

- [ ] 새 영역이 기존 자신의 영역과 1m 이상 겹칠 경우 두 폴리곤이 자동 병합되어 하나의 Territory로 저장된다.
- [ ] 병합 후 기존 Territory 레코드는 삭제되고 새 레코드가 생성된다.
- [ ] Pro: 다른 유저의 영역과 새 영역이 겹칠 경우, 겹치는 면적만큼 기존 유저 영역에서 제거된다.
- [ ] Pro: 영역을 빼앗긴 유저에게 "당신의 영역 XX m²가 침범당했습니다" 푸시 알림이 발송된다.
- [ ] 영역 병합 / 침범 여부는 달리기 종료 직후 결과 화면에 표시된다.

### 기능 2. 챌린지 미션 시스템 (Daily & Weekly Challenges)

**우선순위:** High
**예상 복잡도:** Medium (Free 버전에서도 로컬로 구현 가능)

#### 사용자 스토리

> 나는 달리기를 매일 하고 싶지만 동기부여가 쉽게 떨어진다. 앱이 매일 / 매주 달성 가능한 미션을 제시하고, 완료 시 앱 내 보상(특별 영역 색상, 배지)을 주어 지속적으로 달리게 만들어 주기를 원한다.

#### 기능 상세

- **데일리 챌린지 예시:**
  - "오늘 3km 이상 달리기"
  - "새 영역을 하나 확보하기"
  - "5분 이상 달리기"
- **위클리 챌린지 예시:**
  - "이번 주 총 15km 달리기"
  - "서로 붙어있지 않은 영역 3개 확보하기"
  - "3일 연속 달리기 (스트릭)"
- **보상 시스템:**
  - 챌린지 완료 시 특별 영역 테두리 스타일 잠금 해제
  - 배지 컬렉션 (홈 화면에 표시)
  - 특별 색상 팔레트 잠금 해제
- **구현 포인트:** `RunStats` 엔티티와 `runHistoryProvider` 데이터로 로컬 진행률 계산 가능.

#### 수락 기준 (Acceptance Criteria)

- [ ] 홈 화면에 오늘의 챌린지 카드 1~2개가 표시된다.
- [ ] 챌린지 진행률이 퍼센트 바로 실시간 업데이트된다.
- [ ] 챌린지 완료 시 축하 애니메이션과 함께 보상 내용이 표시된다.
- [ ] 완료된 챌린지는 히스토리 화면에서 달리기 기록과 함께 조회 가능하다.
- [ ] 자정에 데일리 챌린지가 자동 초기화된다.
- [ ] 획득한 배지는 프로필 화면에서 조회 가능하다.

### 기능 3. 영역 히트맵 & 거점 분석 (Territory Heatmap & Hotspot Analysis)

**우선순위:** Medium
**예상 복잡도:** Medium (로컬 데이터만으로 구현 가능)

#### 사용자 스토리

> 나는 지금까지 내가 달린 모든 경로를 지도 위에 열지도(Heatmap)로 한눈에 보고 싶다. 어느 동네를 많이 뛰었는지, 아직 한 번도 달리지 않은 구역이 어디인지 파악해서 다음 달리기 목적지를 계획하고 싶다.

#### 기능 상세

- **히트맵 레이어:** 지도 화면에 기존 TerritoryLayer 위에 토글 버튼으로 ON/OFF. 방문 빈도에 따라 색상 강도 조절.
- **미탐험 구역 표시:** 홈 위치 기준 반경 5km 내 아직 달리지 않은 격자(200m×200m)를 연한 회색으로 표시.
- **월별 영역 타임랩스:** 히스토리 화면에서 월을 선택하면 해당 월 확보 영역이 지도에 애니메이션으로 표시.

#### 수락 기준 (Acceptance Criteria)

- [ ] 지도 화면 우측 상단에 "히트맵" 토글 버튼이 표시된다.
- [ ] 히트맵 ON 시, 모든 과거 달리기 경로가 방문 횟수에 따라 투명도가 다른 색상으로 렌더링된다.
- [ ] 히트맵 레이어와 영역 레이어가 동시에 표시될 수 있다.
- [ ] 설정에서 히트맵 기준 기간(전체 / 최근 30일 / 최근 7일)을 선택할 수 있다.
- [ ] 히트맵 렌더링이 200개 이상의 경로 포인트에서도 60fps를 유지한다.

### 기능 4. 영역 이름 짓기 & 포토 마킹 (Territory Naming & Photo Pinning)

**우선순위:** Medium
**예상 복잡도:** Low~Medium (로컬 구현, 미디어 권한 필요)

#### 사용자 스토리

> 나는 특별히 기억에 남는 달리기로 확보한 영역에 이름을 붙이고 싶다. "한강 노을 구역", "첫 10km 루프" 처럼 개인적인 의미를 담고, 달리기 중 찍은 사진을 GPS 위치에 핀으로 꽂아서 추억을 기록하고 싶다.

#### 기능 상세

- **영역 이름 지정:** `Territory` 엔티티에 `name: String?` 필드 추가. 영역 확보 후 결과 화면에서 이름 입력 가능.
- **포토 핀:** 달리기 중 카메라 버튼 → 사진 촬영 → GPS 좌표와 함께 저장. 지도에서 원형 썸네일 마커로 표시.
- **공유 기능:** 영역 + 사진을 카드 이미지로 합성하여 SNS 공유.

#### 수락 기준 (Acceptance Criteria)

- [ ] Territory에 이름(최대 30자)을 입력하고 수정할 수 있다.
- [ ] 달리기 화면 HUD에 카메라 아이콘 버튼이 표시된다.
- [ ] 지도 화면에서 사진 핀이 소형 원형 썸네일 마커로 표시된다.
- [ ] 마커 탭 시 사진이 전체 화면으로 확대되고 촬영 위치와 날짜가 표시된다.
- [ ] "공유" 버튼 탭 시 영역 폴리곤 + 통계 + 이름이 포함된 카드 이미지가 공유된다.

### 기능 5. 목표 구역 설정 및 달리기 코스 가이드 (Target Zone & Route Planner)

**우선순위:** Medium
**예상 복잡도:** Medium~High

#### 사용자 스토리

> 나는 달리기 전에 지도에서 "오늘은 저 구역을 차지해야지"라고 목표를 정하고, 앱이 선택한 구역을 루프로 달리는 최적 경로를 제안하고 경로 이탈 시 안내해 주기를 원한다.

#### 기능 상세

- **목표 구역 그리기:** 손가락으로 폴리곤을 직접 그려 목표 영역 설정. 경계를 따르는 권장 루프 경로 자동 생성.
- **코스 가이드 모드:** 달리는 동안 권장 경로에서 30m 이상 이탈 시 HUD에 방향 화살표 표시.
- **프리셋 코스:** 거리 입력 시 현재 위치 기준 정사각형/삼각형/나선형 루프 자동 생성.
- **구현 포인트:** `GeoUtils.polygonContainsPoint()` 활용하여 경로 이탈 감지.

#### 수락 기준 (Acceptance Criteria)

- [ ] 지도 화면에서 손가락으로 폴리곤을 그릴 수 있다 (최소 3개 꼭짓점).
- [ ] 그려진 목표 구역 경계를 따르는 권장 루프 경로가 자동 생성된다.
- [ ] 가이드 모드에서 경로에서 30m 이상 이탈 시 HUD에 방향 화살표가 표시된다.
- [ ] 달리기 완료 후 결과 화면에 "목표 대비 달성률 XX%"가 표시된다.
- [ ] 거리 입력 시 현재 위치 기준 정사각형 루프 코스가 자동 생성된다.

## 개발 권장 순서

1. **챌린지 미션** — 서버 없이 즉시 구현 가능, DAU 향상 효과 즉각적
2. **영역 이름 짓기 & 포토 마킹** — 감성적 연결고리 형성, 공유 바이럴 기대
3. **영역 합산 (로컬 병합)** — `GeoUtils.mergePolygons` 이미 존재, 공수 최소
4. **영역 히트맵** — 기존 데이터 재활용, 장기 리텐션 기여
5. **목표 구역 & 코스 가이드** — 신규 달리기 행동 패턴 유도, 가장 혁신적
